Explain everything you just did in the simplest English possible as a **single paragraph**. Pretend you're talking to someone who has never coded before.

- One paragraph only — no headers, no bullets inside the explanation. Keep it flowing.
- No jargon — if you must use a technical word, explain it in parentheses like you're defining it for a kid
- Use real-world analogies (e.g. "a database is like a spreadsheet", "an API is like a waiter taking your order")
- Short sentences, but keep them in one paragraph
- Explain the "why" before the "what" — why did we need this change, then what we actually did
- Group changes by feature, not by file — weave them together in the paragraph
